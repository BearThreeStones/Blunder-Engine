# ClassDB is Clang-exported with PtrCall primary and Variant secondary

Editor tooling and future script bindings share one **ClassDB**. Exported types are deliberately marked in C++; a Clang AST–based generator (MSVC remains the engine compiler) produces ClassDB registration and a language-neutral **API Blueprint** for binding generators — C# must not scrape C++ headers or hold C++ member pointers. Dynamic calls use **PtrCall** (typed slots, low/zero allocation) for script methods and lifecycle dispatch; a small engine **Variant** covers Inspector property get/set and serialization glue only. Lifecycle overrides (Ready/Tick) use a **per-exported-type** replaceable dispatch table invoked via PtrCall on Objects that have a Script Peer — not per-instance native function pointers. Object-facing properties are the script/editor API; ECS fields are reached through projected accessors on that surface. Rejected alternatives: dual editor-vs-script metadata stores, pure macro/template static registration as the long-term source of truth, Variant-only interop, and exposing C++ layouts across the managed boundary.
