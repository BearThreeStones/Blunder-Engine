# Editor History MVP covers transform, spawn, and soft-delete

The first Editor History milestone ships Document History plumbing plus these Commands: entity transform commits (gizmo drag end, Translate Modal confirm, Inspector TRS commit), entity spawn, and entity delete. Delete is a **soft delete** (tombstone): the entity keeps its dense `EntityId`, is hidden/disabled in the editable view, and can be restored by undo; Save/`exportToScene` omits tombstones so disk matches the editable document. Commands seal only at interaction boundaries (no timer coalesce). Stack depth is a fixed limit (default 100). Undo/Redo are exposed via Ctrl+Z, Ctrl+Y and Ctrl+Shift+Z, and Edit menu items. Explicitly out of scope: generational/free-list EntityId allocation, physical mid-vector erase, reparent/hierarchy Commands beyond what spawn/delete imply, ObjectId targeting, branched history, persisting tombstones in `.scene.asset`, and restoring undo stacks across `openScene`. Rejected alternative: transform-only or plumbing-only MVP that ships no user-visible spawn/delete undo.
