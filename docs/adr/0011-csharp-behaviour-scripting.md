# C# Behaviours are the gameplay scripting model

Blunder's sole first-class **gameplay scripting language** is **C#**. Gameplay scripts are Unity-style **Behaviours**: an **Object** hosts an ordered list of zero or more Behaviours (duplicate types allowed); each Behaviour has one **Script Peer** and a stable **BehaviourId** (not a list index). Behaviours are not Objects and not ECS Components; they reach engine state through the host Object and sibling queries (`GetBehaviour` / `GetBehaviours`). Ready/Tick run once per Behaviour in list order via PtrCall. The runtime host is in-process **CoreCLR** (`nethost` / hostfxr); ALC hot reload comes later. Editor and managed ScriptHost share **one process ObjectDB** via a native-registered C-ABI function-pointer table (`RegisterNativeAbi` / `blunder_native_abi_fill_from_process`). **Play Mode** runs the .NET host inside the **Player** process (`engine_player`); Edit Mode does not start a host for normal authorship (ADR 0014) — `BLUNDER_DOTNET_SCRIPTS` remains a debug/test escape hatch, not the product Play path. After the Reflection kernel: **.NET host MVP** (load one Project assembly, attach Behaviours, lifecycle — no scene persistence / Inspector UX / hot reload), then a **Behaviour serialization slice** (shipped: scene JSON `behaviours` with type + BehaviourId + optional bool/number/string property bag; Object bind + id restore on load; mount peers when DotNetHost is running; export type/id from bound Objects — still no Inspector UX / ALC hot reload; Play UI per ADR 0014; depends on single-ObjectDB NativeAbi from `unify-script-objectdb`). Projects keep C# under a root **`Scripts/`** folder (`net10.0`); Create scaffolds a minimal `.csproj` that references a generated **Engine API assembly** (`Blunder.Api`) beside the editor. Dev builds: editor invokes `dotnet build` when Scripts are dirty before Play; shipping uses a separate publish path. This **amends** ADR 0003's "at most one Script Peer" clause. Rejected alternatives: multi-language official scripting; Godot-style one-script-per-Object as the product rule; Behaviour-as-Object subclass; scripts on ECS Components; Mono host; out-of-process `dotnet` IPC as the shipping **script** model (Play control IPC between editor and Player is separate — ADR 0014); bundling hot reload into the first host milestone; storing gameplay `.cs` under `Assets/`.
