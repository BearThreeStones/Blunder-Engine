# Dual-track Object identity layer and ECS data layer

Blunder keeps a dual runtime model: **Object** is the ClassDB citizen for naming, Scene Tree hierarchy, serialization entry, lifecycle hooks, and at most one Script Peer; **ECS World** stores recyclable entities and components for hot data and systems. Components are not Objects and do not host GCHandles. An Object may optionally bind one `EntityId`; simulation-only entities need no Object; identity-only Objects need no Entity. The Scene Tree lives only on Objects and is projected into ECS transform parenting when an Entity exists. Public cross-language identity is **ObjectId**, not dense `EntityId` and not raw C++ pointers. Writing a spatial property on an Object with no Entity lazily materializes an Entity plus Transform and binds it; destroying the Object despawns that Entity. Rejected alternatives: pure Object tree (fights current EntityId/`SceneInstance`), pure ECS with scripts on components (hot-reload and Inspector identity collapse), mandatory 1:1 Object↔Entity, dual-written parent links, and per-component Script Peers.
