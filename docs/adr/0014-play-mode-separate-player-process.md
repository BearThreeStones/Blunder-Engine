# Play Mode runs in a separate Player process

**Play Mode** is a try-the-game session owned by a dedicated **Player** executable (`engine_player`: thin main over the shared engine runtime), not an in-editor simulation that mutates the authorship `SceneInstance`. The editor stays in **Edit Mode**, may keep editing, and drives at most one **Play session** via Play / Pause / Stop over a **local IPC control channel** (pause skips Behaviour Tick while the window stays viewable; stop and closing the Player window end the process). Play loads the **saved** active scene asset (dirty documents get Save and Play / play last saved / cancel); the editor builds Project `Scripts/` when dirty before spawn. Live sync of edits or ALC hot-reload into a running Player is out of this first Play UI slice. Edit Mode does not start a .NET script host for normal authorship — gameplay peers and Tick run in the Player (env-gated editor host remains debug/test only). Rejected for this product shape: in-editor Play in the authorship viewport; same-process second Game window as the Play boundary; reusing `engine_editor --play` as the long-term Player; auto-save-only or silent dirty Play; requiring live hot-sync before Play UI ships.
