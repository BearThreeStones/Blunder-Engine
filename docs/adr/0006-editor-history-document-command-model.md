# Editor History is document-scoped command undo

Editor undo/redo is **Editor History**: a linear **Document History** stack owned by the active editable scene (`activeScenePath` / active `SceneInstance`), not a runtime/gameplay rewind system. Units of work are **Editor Commands** with `undo`/`redo`; continuous interactions (gizmo drag, Translate Modal) seal one Command at confirm. Commands target `EntityId` in v1; each Command stores before/after selection snapshots restored on undo/redo. Dirty state tracks divergence from the last successful save via a history baseline cursor. New Commands after undo truncate the redo tail. Opening another scene clears/replaces that Document History. Rejected alternatives: global cross-document undo, Memento-only snapshots without Command boundaries, branched undo trees, ObjectId as the v1 command target.
